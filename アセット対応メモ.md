# 画像アセット対応メモ（手作業が必要）

デザインレビューのうち、**画像ファイルの作り直しが必要な項目**をまとめます。コードでは完結できないため、画像を差し替える手作業をお願いします。差し替え先のコードはすでに対応済みなので、**同じファイル名で置き換えるだけ**で反映されます。

対象レビュー: [`デザインレビュー.md`](./デザインレビュー.md) の F / M（H は SVG で対応済み）

---

## F. ロゴの背景を透過にする ✅ 対応済み

- 対象ファイル: `app/assets/images/logo.png`
- **対応内容**: ユーザー提供の新ロゴ画像を採用。提供画像は背景が不透明（白／市松模様の焼き込み）だったため、ImageMagick の四隅フラッドフィル（連結した背景のみ透過化・キャラ内部の白は保持）で透過PNG化し、`logo.png` を差し替え済み。LP と about ページで、サイト背景（ミント）に自然に溶け込むことを確認。
- 参考コマンド（再加工が必要な場合）:
  ```sh
  magick 元画像.png -alpha set -fuzz 12% -fill none \
    -draw "color 0,0 floodfill" -draw "color 1023,0 floodfill" \
    -draw "color 0,1023 floodfill" -draw "color 1023,1023 floodfill" \
    app/assets/images/logo.png
  ```
- 残メモ（任意・同じトーンで揃えると全体が統一される）:
  - `app/assets/images/apple-touch-icon.png`（ホーム画面用）
  - `public/ogp.jpg`（SNS シェア画像）

## M. ファビコンを低解像度でも分かる専用デザインに

- 対象ファイル: `app/assets/images/favicon.png`
- 現状: ロゴをそのまま縮小しているため、16px ではキャラや家族の要素がつぶれ「緑の丸に白いモヤ」に見え、何のサービスか判別できない。
- **やること: ロゴの縮小版ではなく、ファビコン専用にデザインし直す**。
  - 原則: ①1つの概念だけに絞る ②太い線・大きな面・高コントラスト ③モチーフを画面いっぱいに。
  - モチーフ案:
    - サービス名「おくす**リング**」にかけた **リング（輪）＋ 錠剤／カプセル** の組み合わせ。名前と意味がつながり覚えやすい。
    - もしくは **カプセル1粒** だけ、または **キャラの顔だけ** を大きく単純化。
  - 色は今のミント／グリーンを踏襲するとブランドの一貫性を保てる。
- 確認方法: 実際に **16×16 / 32×32 に縮小して視認**し、一目で分かるかチェック。
- 作り分けの推奨:
  - `favicon.png`（タブ用・小）… 思い切って単純化。
  - `apple-touch-icon.png`（ホーム画面用・180px 程度）… 少し情報量を足してよい。

## H. チェックアイコン（対応済み・参考）

- 服薬状況の「済/未」は、環境依存の絵文字からインラインSVG（Material Symbols 風の丸みアイコン）へ**コード側で変更済み**（`app/helpers/application_helper.rb` の `medication_status_badge`）。外部フォント依存なし。
- 残りの**装飾**絵文字（タイトルの💊、LPの💊✅👨‍👩‍👧、家族の👤、招待の🔗💌）は、ブランドの雰囲気づくりの装飾として現状のままにしている（`aria-hidden` 付与済み）。
  - もし将来これらも図版として統一したい場合は、同じ要領でインラインSVG化するか、SVGスプライトを用意すると保守しやすい。
